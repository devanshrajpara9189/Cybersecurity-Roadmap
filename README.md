# 📅 Day 1: Cybersecurity Basics

## 🎣 Phishing Attack

Phishing is a social engineering attack where attackers try to steal sensitive information through:

- Email
- SMS
- Fake websites

### Prevention Tools:

- **VirusTotal**
- Multi-Factor Authentication
- Strong Passwords

---

## 💰 Ransomware

A malware attack where attackers encrypt data and demand money (ransom) for recovery.

### Prevention:

- Regular backups
- Offline backups
- Security updates

---

# 🦠 Malware

Malware is malicious software designed to harm systems.

Types:

- **Virus**
- **Worm**
- **Trojan Horse**
- **Spyware**
- **Keylogger**
- **Ransomware**

---

# 🕵️ Man In The Middle Attack (MITM)

An attacker secretly intercepts communication between two systems.

Examples:

- Packet sniffing
- Session hijacking

### Prevention:

- HTTPS websites
- VPN on public WiFi
- Encryption

---

# 🔺 CIA Triad

The foundation of cybersecurity.

## Confidentiality

- Data access only by authorized users
- Protection methods:
  - Passwords
  - Two-Factor Authentication

## Integrity

- Data should remain accurate and trustworthy

Protection:

- Hashing
- Digital Signatures

## Availability

- Systems and data should be available when needed

Protection:

- Backup plans
- Anti-DDoS protection

# ⚠️ Cybersecurity Terminology

## Vulnerability

A weakness in software, hardware, or systems.

## Threat

A possible cyber attack or danger.

## Risk

Possible impact caused by exploiting vulnerabilities.

## Exploit

Technique used to take advantage of vulnerabilities.

## Payload

Code or action performed after exploitation.

# 👥 Security Teams

# 🔴 Red Team

Offensive security team.

Responsibilities:

- Simulates attacks
- Finds vulnerabilities
- Tests security controls
- Exploits weaknesses

---

# 🔵 Blue Team

Defensive security team.

Responsibilities:

- Protect systems
- Monitor threats
- Detect attacks
- Improve security

---

# 🟣 Purple Team

Combination of Red Team + Blue Team.

Responsibilities:

- Threat hunting
- Security monitoring
- Improving detection
- Testing defenses

Workflow:

1. Plan attack scenario
2. Red team performs attack
3. Blue team detects and responds
4. Analyze results
5. Improve defenses
6. Retest security
