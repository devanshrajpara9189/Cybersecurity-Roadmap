# 🌐 IP Address

An **IP address (Internet Protocol Address)** is a unique identifier assigned to a device on a network.

Purpose:

- Device identification
- Communication between devices

Types:

## IPv4

Example:
192.168.1.10


## IPv6

Example:
2001:db8::1


---

# 💻 Network

A network is a group of connected devices that communicate and share resources.

Examples:

- Home network
- Office network
- Internet

Network components:

- Computers
- Routers
- Switches
- Servers

---

# 🔐 VPN (Virtual Private Network)

A VPN creates a secure encrypted tunnel between a device and a network through the internet.

Benefits:

- Protects data transmission
- Provides privacy
- Secures public WiFi connections
- Allows secure remote access

Example:

Employee connecting securely to company network from home.

---

# 🌍 Proxy

A proxy server acts as an intermediate system between a user device and the internet.

Working:

User → Proxy Server → Internet

Functions:

- Hide original IP address
- Filter web traffic
- Control internet access

Example:

- Proxyium

---

# 💥 Exploit

An exploit is a technique or code used to take advantage of a vulnerability.

Purpose:

- Gain unauthorized access
- Execute commands
- Access restricted resources

Example:

Using a software vulnerability to gain system access.

---

# 📦 Payload

A payload is the code or action executed after successfully exploiting a vulnerability.

Examples:

- Opening a remote connection
- Running commands
- Collecting information

In penetration testing, payloads are used to test security controls in authorized environments.

---

# 🔑 Authentication

Authentication verifies the identity of a user or system.

Examples:

- Username and password
- Fingerprint
- Multi-Factor Authentication (MFA)

---

# 🔐 Authorization

Authorization determines what actions or resources an authenticated user can access.

Example:

A normal user cannot access administrator settings.

---

# 🔒 Encryption

Encryption converts readable data (plaintext) into unreadable data (ciphertext).

Purpose:

- Protect confidentiality
- Prevent unauthorized access

---

# #️⃣ Hashing

Hashing converts data into a fixed-length value.

Uses:

- Password storage
- File integrity checking
- Digital verification

Examples:

- SHA-256
- SHA-512

---

# 📡 Protocol

A protocol is a set of rules that allows devices to communicate over a network.

Examples:

- HTTP
- HTTPS
- TCP
- UDP
- DNS
- SSH

---

# 🖥️ Server

A server is a computer system that provides services, resources, or data to other devices.

Examples:

- Web server
- Database server
- File server

---

# 📝 Patch

A patch is a software update released to fix bugs and security vulnerabilities.

Purpose:

- Improve security
- Fix errors
- Improve performance

---

# 🕵️ Zero-Day Vulnerability

A vulnerability unknown to the vendor or without an available patch.

Attack using this vulnerability is called a:

**Zero-Day Attack**

---

# 📊 CVE (Common Vulnerabilities and Exposures)

A public database containing information about known cybersecurity vulnerabilities.

Example:

CVE-2024-XXXX

---

# 🧩 CWE (Common Weakness Enumeration)

A list of common software and hardware weaknesses.

Examples:

- Buffer Overflow
- SQL Injection
- Improper Authentication

---

# 🛰️ SOC

Security Operations Center responsible for:

- Monitoring
- Detection
- Investigation
- Incident response

---

# ⚔️ Incident Response

The process of handling cybersecurity incidents.

Steps:

1. Preparation
2. Detection
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned
