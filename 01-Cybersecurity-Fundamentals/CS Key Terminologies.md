# 🔐 Cybersecurity Terminology

Understanding basic cybersecurity terms is important for learning security concepts, attacks, and defense mechanisms.

---

# 🦠 Virus

A **virus** is a type of malicious software (malware) that attaches itself to files or programs and spreads when the infected file is executed.

Purpose:

- Steal sensitive information
- Damage files
- Modify system data

Protection:

- Antivirus software
- Regular updates
- Avoid suspicious files

---

# ⚠️ Threat

A **threat** is any possible danger that can exploit a vulnerability and cause harm to a system, network, or data.

Examples:

- Malware attacks
- Phishing attacks
- Data breaches
- Unauthorized access

---

# 📉 Risk

**Risk** is the possibility of damage or loss when a threat exploits a vulnerability.

Risk depends on:

- Threat
- Vulnerability
- Impact

Example:

A weak password (vulnerability) can be exploited by an attacker (threat), causing account theft (impact).

---

# 🔓 Vulnerability

A **vulnerability** is a weakness or security flaw in hardware, software, network, or processes.

Examples:

- Outdated software
- Weak passwords
- Misconfigured systems
- Security bugs

---

# 🔍 Vulnerability Assessment

Vulnerability Assessment is the process of identifying, analyzing, and reporting security weaknesses in a system.

Steps:

1. Discover vulnerabilities
2. Analyze severity
3. Generate reports
4. Recommend fixes

Common Severity Levels:

- Critical
- High
- Medium
- Low

Tools:

- Nessus
- OpenVAS
- Qualys

---

# 🛡️ Penetration Testing

Penetration Testing is an authorized security test performed to identify and exploit vulnerabilities before attackers can use them.

Goals:

- Find security weaknesses
- Test defenses
- Improve security

Stages:

1. Reconnaissance
2. Scanning
3. Enumeration
4. Exploitation
5. Reporting

---

# 🔥 Firewall

A firewall is a security system that monitors and controls incoming and outgoing network traffic.

Functions:

- Allow trusted connections
- Block unauthorized access
- Filter network traffic

Types:

- Network Firewall
- Host Firewall
- Application Firewall

Example:

A firewall acts like a security guard between a private network and the internet.

---

# 🚨 IDS (Intrusion Detection System)

IDS monitors network traffic and detects suspicious activities.

Functions:

- Analyze network behavior
- Detect threats
- Generate alerts

IDS mainly focuses on **detection**.

---

# 🛑 IPS (Intrusion Prevention System)

IPS detects and prevents malicious activities automatically.

Functions:

- Monitor traffic
- Block attacks
- Prevent unauthorized access

IPS focuses on **detection + prevention**.

---

# 🌐 IP Address

An **IP address (Internet Protocol Address)** is a unique identifier assigned to a device on a network.

Purpose:

- Device identification
- Communication between devices

Types:

## IPv4

Example:
192.168.1.10


## IPv6

Example:
2001:db8::1


---

# 💻 Network

A network is a group of connected devices that communicate and share resources.

Examples:

- Home network
- Office network
- Internet

Network components:

- Computers
- Routers
- Switches
- Servers

---

# 🔐 VPN (Virtual Private Network)

A VPN creates a secure encrypted tunnel between a device and a network through the internet.

Benefits:

- Protects data transmission
- Provides privacy
- Secures public WiFi connections
- Allows secure remote access

Example:

Employee connecting securely to company network from home.

---

# 🌍 Proxy

A proxy server acts as an intermediate system between a user device and the internet.

Working:

User → Proxy Server → Internet

Functions:

- Hide original IP address
- Filter web traffic
- Control internet access

Example:

- Proxyium

---

# 💥 Exploit

An exploit is a technique or code used to take advantage of a vulnerability.

Purpose:

- Gain unauthorized access
- Execute commands
- Access restricted resources

Example:

Using a software vulnerability to gain system access.

---

# 📦 Payload

A payload is the code or action executed after successfully exploiting a vulnerability.

Examples:

- Opening a remote connection
- Running commands
- Collecting information

In penetration testing, payloads are used to test security controls in authorized environments.

---

# 🔑 Authentication

Authentication verifies the identity of a user or system.

Examples:

- Username and password
- Fingerprint
- Multi-Factor Authentication (MFA)

---

# 🔐 Authorization

Authorization determines what actions or resources an authenticated user can access.

Example:

A normal user cannot access administrator settings.

---

# 🔒 Encryption

Encryption converts readable data (plaintext) into unreadable data (ciphertext).

Purpose:

- Protect confidentiality
- Prevent unauthorized access

---

# #️⃣ Hashing

Hashing converts data into a fixed-length value.

Uses:

- Password storage
- File integrity checking
- Digital verification

Examples:

- SHA-256
- SHA-512

---

# 📡 Protocol

A protocol is a set of rules that allows devices to communicate over a network.

Examples:

- HTTP
- HTTPS
- TCP
- UDP
- DNS
- SSH

---

# 🖥️ Server

A server is a computer system that provides services, resources, or data to other devices.

Examples:

- Web server
- Database server
- File server

---

# 📝 Patch

A patch is a software update released to fix bugs and security vulnerabilities.

Purpose:

- Improve security
- Fix errors
- Improve performance

---

# 🕵️ Zero-Day Vulnerability

A vulnerability unknown to the vendor or without an available patch.

Attack using this vulnerability is called a:

**Zero-Day Attack**

---

# 📊 CVE (Common Vulnerabilities and Exposures)

A public database containing information about known cybersecurity vulnerabilities.

Example:

CVE-2024-XXXX

---

# 🧩 CWE (Common Weakness Enumeration)

A list of common software and hardware weaknesses.

Examples:

- Buffer Overflow
- SQL Injection
- Improper Authentication

---

# 🛰️ SOC

Security Operations Center responsible for:

- Monitoring
- Detection
- Investigation
- Incident response

---

# ⚔️ Incident Response

The process of handling cybersecurity incidents.

Steps:

1. Preparation
2. Detection
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned
